# 瓦片尺寸对比测试流程

## 目标

测试不同尺寸的像素艺术瓦片在游戏中的显示效果，选择最佳尺寸。

- 16x16 (原始设计)
- 64x64 (放大4倍)
- 128x128 (放大8倍)

---

## 测试策略

像素艺术游戏的关键点：
- **像素感** - 瓦片放大后，像素点是否清晰可见
- **细节保留** - 图案纹理是否还能辨认
- **风格一致性** - 不同尺寸是否保持统一风格
- **性能** - 内存占用是否合理

---

## 生成流程

### 第一步：生成原始瓦片 (16x16)

```bash
cd resources/tilesets
python generate_tiles.py
```

生成文件：
- `generated_tiles/floor_lobby.png` (16x16)
- `generated_tiles/floor_common.png` (16x16)
- ...其他瓦片

### 第二步：生成 64x64 测试瓦片

修改 `generate_tiles.py` 中的 `TILE_PIXELS`：

```python
TILE_PIXELS = 64  # 改为64
```

```bash
cd resources/tilesets/generated_tiles
mkdir -p test_64x64
python ../generate_tiles.py  # 需要修改脚本支持参数
```

或使用批量处理脚本：

```python
# resize_test.py
from PIL import Image
from pathlib import Path

SRC_DIR = Path("generated_tiles")
DST_64 = Path("generated_tiles/test_64x64")
DST_128 = Path("generated_tiles/test_128x128")

DST_64.mkdir(exist_ok=True)
DST_128.mkdir(exist_ok=True)

for png in SRC_DIR.glob("floor_*.png"):
    img = Image.open(png).convert("RGBA")

    img64 = img.resize((64, 64), Image.Resampling.NEAREST)
    img64.save(DST_64 / png.name)

    img128 = img.resize((128, 128), Image.Resampling.NEAREST)
    img128.save(DST_128 / png.name)

print("Done: 生成 64x64 和 128x128 测试瓦片")
```

### 第三步：创建对比测试场景

在 Godot 中创建测试场景：

```
测试场景结构：
res://scenes/test/
├── test_tiles_16.tscn    # 原始16x16瓦片
├── test_tiles_64.tscn    # 64x64瓦片
├── test_tiles_128.tscn   # 128x128瓦片
└── test_compare.tscn      # 并排对比场景
```

### 第四步：配置 TileSet

为每个尺寸创建独立的 TileSet：

```bash
cd resources/tilesets/generated_tiles/test_64x64
python ../generate_tileset.py  # 需修改输出路径
```

### 第五步：Godot 内对比测试

1. 创建3个 TileMap 节点，使用不同 TileSet
2. 用相同地图数据填充
3. 全屏预览对比

---

## 判断标准

| 指标 | 16x16 | 64x64 | 128x128 |
|------|-------|-------|---------|
| 像素清晰度 | 最清晰 | 中等 | 可能模糊 |
| 细节可读性 | 简单图案 | 较多细节 | 最丰富 |
| 风格像素感 | 强 | 中等 | 弱 |
| 内存占用 | 最低 | 4倍 | 16倍 |
| 适用场景 | 经典像素风 | 平衡方案 | 高清像素风 |

---

## 推荐工作流程

```
1. 先生成一套 64x64 瓦片 (平衡方案)
2. 再生成一套 128x128 瓦片 (高清测试)
3. 在游戏中实际对比
4. 确定最佳尺寸
5. 批量生成选定尺寸的瓦片
```

## 批量生成脚本修改建议

修改 `generate_tiles.py` 支持尺寸参数：

```python
import argparse

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--size", type=int, default=64, choices=[16, 32, 64, 128])
    parser.add_argument("--output", type=str, default="generated_tiles")
    args = parser.parse_args()

    global TILE_PIXELS
    TILE_PIXELS = args.size
    # ...后续生成逻辑
```

使用示例：
```bash
python generate_tiles.py --size 64 --output generated_tiles/test_64x64
python generate_tiles.py --size 128 --output generated_tiles/test_128x128
```
